AgentLayerWeb

The interaction layer for AI-native web applications.

AgentLayerWeb is a headless framework that adds a semantic interaction layer to your web application, allowing AI agents to understand, navigate, and operate your app without relying on DOM parsing, screenshots, or brittle browser automation.

Core Insight

HTML gives structure to web pages for human eyes; ARIA makes them accessible. AgentLayerWeb provides the typed structure, permissions, error recovery guidelines, and manifests that AI agents need to interact with websites natively.

Core Philosophy

Modern web applications are built for human cognitive capabilities. AI agents must reverse engineer applications by parsing classnames, DOM hierarchies, and visual screenshots. This is brittle. A small CSS change or button layout shift can break an agent.

AgentLayerWeb solves this by defining Intent Over Appearance. Instead of guessing what a button does, developers explicitly describe actions, requirements, and state transitions using strongly-typed metadata.

React Installation

Install the core and React primitives from our packages registry: 

pnpm add @agentlayerweb/core @agentlayerweb/react

Core Primitives

Annotate your React application with semantic components. These are completely headless, do not modify layout styling, and inject `data-agent-*` attributes into child elements automatically: 

import { AgentApp, AgentPage, AgentSection, AgentAction } from '@agentlayerweb/react';
 
export default function App() {
  return (
    <AgentApp name="Documentation Portal" version="1.0">
      <AgentPage id="docs_home" title="Documentation Home">
        <AgentSection id="api_reference" title="API Reference">
          <AgentAction 
            id="copy_code" 
            intent="copy" 
            priority="primary"
            description="Copies code block to clipboard"
          >
            <button onClick={copyCode}>Copy Code</button>
          </AgentAction>
        </AgentSection>
      </AgentPage>
    </AgentApp>
  );
}

All 28 Semantic Primitives

Use these primitives to annotate your layouts. Spreading them (React) or binding them (Vue/Svelte) attaches standard data-agent-* selectors directly to the rendered elements.

Primitive Category Usage Example
agent.app Application agent.app({ name: 'Portal', version: '1.0' })
agent.page Navigation agent.page({ id: 'dashboard', title: 'Dashboard' })
agent.section Layout agent.section({ id: 'billing' })
agent.action Interaction agent.action({ id: 'checkout', priority: 'primary' })
agent.form Data Input agent.form({ purpose: 'Submit Feedback' })
agent.field Data Input agent.field({ name: 'price', type: 'number' })
agent.entity Data agent.entity({ type: 'User', id: 'usr-1' })
agent.workflow Workflow agent.workflow({ id: 'checkout_flow' })
agent.step Workflow agent.step({ id: 'payment', current: true })
agent.dialog Interface agent.dialog({ purpose: 'confirm_delete' })
agent.error State agent.error({ code: 'INVALID_PRICE' })

Other primitives include: collection, table, search, filter, sort, loading, success, status, emptyState, breadcrumb, sidebar, tabs, command, shortcut, tooltip.

Local DOM Validation

Run developer audits dynamically in the browser to detect duplication, hierarchy leaks, or invalid nesting of agent roles: 

import { validateAgentLayerWeb } from '@agentlayerweb/core';

// Crawls the document and logs errors in console
validateAgentLayerWeb();
